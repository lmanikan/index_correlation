# Final Implementation Plan: Refined ETL and DTO Analytics

## Objective
Standardize the `src/` directory using a configuration-driven ingestion flow and a DTO-based transformation layer.

## Key Files & Context
- **Core:** `src/core/models.py` (Unified models + DTOs).
- **Analytics:** `src/analytics/` (Renamed from `transforms/`).
- **Extraction:** `src/extraction/` (New, renamed from `connectors/`).
- **Config:** `src/config/index_config.py` and `indexes.yaml` (Adding `quantities`).

## Implementation Steps

### Phase 1: Core Model Unification & DTOs
1.  **Update `src/core/models.py`:**
    - Consolidate all enums (`DataSourceType`, `WeightType`, `VolType`).
    - Define `TransformationInput` DTO.
    - Merge all models from `data_models.py` (Index, IndexWeights, DataPackage, TrialResults, etc.).
    - Implement `DataPackage.to_transformation_inputs()` to facilitate granular transformations.
2.  **Clean Up:** Update `src/core/__init__.py` and delete `src/core/data_models.py`.

### Phase 2: Analytics Refactoring (The Engine)
1.  **Rename Directory:** Move `src/transforms/` to `src/analytics/`.
2.  **Refactor `BaseQuantity`:** Update the `compute` method to accept `TransformationInput` DTO.
3.  **Refactor `AnalyticsEngine`:**
    - Update `_compute_term` to iterate over DTOs generated by `DataPackage`.
    - Support dynamic dispatch based on the `quantities` specified in the `Index` config.

### Phase 3: Extraction Refactoring (The Ingestion)
1.  **Rename Directory:** Move `src/connectors/` to `src/extraction/`.
2.  **Refactor `DataLoader`:**
    - Implement the sequential ingestion logic: `Index -> Components -> Identification -> Fetching -> Assembly`.
    - Ensure it uses the updated `Index` model which now includes `quantities`.

### Phase 4: Configuration Update
1.  **Update `indexes.yaml`:** Add `quantities` list to each index definition.
2.  **Update `src/config/index_config.py`:** Ensure it correctly loads the new fields.

### Phase 5: Storage and Global Import Fix
1.  **Move Writers:** Move `src/extraction/results_writer.py` (previously in connectors) to `src/storage/interface.py`.
2.  **Global Search & Replace:** Systematic update of all `src.*` imports.

## Verification & Testing
1.  **DTO Check:** Verify `DataPackage` can produce valid `TransformationInput` objects.
2.  **Engine Check:** Verify `AnalyticsEngine` computes all requested quantities for a multi-term index.
3.  **End-to-End Dry Run:** Use a mock extractor to verify the flow from `Index Config` to `TrialResults`.
